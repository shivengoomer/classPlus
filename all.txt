classPlus / ClassPilot Project Overview
======================================

Project summary
---------------
classPlus, also called ClassPilot in existing documentation, is an AI-powered assessment and classroom workspace for teachers and students. Teachers can create curriculum-aligned assignments, manage class groups, assign assessments, grade submissions, inspect analytics, and export or store learning materials. Students can join classes, view assigned work, submit answers, upload handwritten sheets, generate AI practice quizzes, review scores, and use an AI tutor experience.

Technology stack
----------------
Frontend:
- Next.js 14 App Router
- React 18
- TypeScript
- Tailwind CSS
- Framer Motion
- Zustand for client state
- Clerk for teacher authentication
- Lucide React icons

Backend:
- Node.js
- Express
- TypeScript
- MongoDB with Mongoose
- Redis and BullMQ for background assignment generation jobs
- WebSocket server for live job updates
- Groq API for AI generation and grading
- PDFKit, pdf-parse, and mammoth for document parsing/export workflows
- Clerk Express middleware for authenticated teacher API routes

Main folders
------------
backend/src:
- app.ts: Express app setup, middleware, and API route mounting.
- server.ts: HTTP and WebSocket server entry point.
- config: MongoDB, Redis, and environment setup.

- models: Mongoose schemas for assignments, groups, students, submissions, practice quizzes, notifications, libraries, templates, and users.
- routes: Express route definitions.
- services: AI, parser, and PDF service logic.
- queues and workers: BullMQ assignment generation queue and worker.
- websocket: WebSocket status broadcasting.

frontend/src:
- app: Next.js App Router pages.
- components/layout: App shell, sidebar, top bar, mobile bottom nav, notifications, and auth layout.
- components/create: Assessment builder UI.
- components/assignments: Assignment cards, grid, context menu, and empty state.
- components/result: Generated exam/result rendering.
- components/shared: Reusable UI primitives such as badges, buttons, logo, progress, toast, and steppers.
- lib: Teacher API client, student API client, validators, and utilities.
- store: Zustand stores for assignments, jobs, notifications, profile, templates, forms, toasts, and student session.
- types: TypeScript data types.

Teacher workspace features
--------------------------
Dashboard:
- Shows teacher overview.
- Loads assignments and profile data.
- Provides quick-start assessment templates.
- Shows active and pending assessment counts.
- Links into create and assessment workflows.

Classrooms / Groups:
- Teachers create class groups with name, grade, subject, rubric, and roster students.
- Each group has a class code for student joining.
- Teachers can add, edit, and delete groups.
- Teachers can assign completed assessments to groups.
- Teachers can inspect group submissions.
- Teachers can manually grade or update submissions.
- Teachers can upload a student paper into a submission flow.

- Shows assignment status.
- Supports viewing completed results.

- Supports question types such as MCQ, short answer, long answer, true/false, and fill in the blank.

- Sends multipart form data to the backend.
- Creates a queued generation job and navigates to progress/status.

Progress Center:
- Tracks background assignment generation by job ID.
- Uses WebSocket updates to reflect processing progress.
- Routes to the generated result when complete.

Generated Result View:
- Displays generated exam/worksheet content.
- Shows question sections, marks, answer keys, and student-facing paper structure.

Library:
- Stores uploaded documents and exported/generated resources.
- Supports folders and library item deletion.

AI Toolkit:
- Workspace area for curriculum and teacher support utilities.
- Connected in navigation and command menu.

Settings and Billing:
- Settings page stores school/profile details.
- Billing page exists in the workspace navigation.
- Profile data feeds the teacher shell and school branding.

Notifications:
- Backend stores teacher notifications.
- Frontend shows unread count and notification panel.
- Supports mark-read, mark-all-read, delete, and clear-all API actions.

Student Reports feature
-----------------------
Purpose:
Student Reports is a teacher analytics dashboard inside the teacher workspace. It aggregates real data from class groups, registered students, assigned homework submissions, AI practice quizzes, topic performance, score growth, and AI feedback.

Navigation:
- Sidebar item: Student Reports, path /reports.
- Mobile drawer item: Student Reports, path /reports.
- Top bar breadcrumb: Workspace / Student Reports.
- Command menu action: Go to Student Reports.

Backend files:
- backend/src/controllers/report.controller.ts
- backend/src/routes/report.routes.ts
- backend/src/app.ts mounts /api/reports

Backend endpoints:
- GET /api/reports/students
  Returns registered students associated with the authenticated teacher's groups.
  Includes student name, email, registered status, enrolled groups, assignments assigned, assignments completed, average homework score, practice quiz count, and average practice score.

- GET /api/reports/student/:email
  Returns a detailed analytics report for one registered student by email.
  Enforces teacher ownership by checking the student's credential group IDs against the authenticated teacher's groups.
  Includes group enrollments, weak topics, strong topics, chronological score timeline, struggle notes, homework logs, and practice logs.

Student Reports aggregation rules:
- Registered students come from StudentCredential records linked to the teacher's group IDs.
- Homework completion is based on StudentSubmission records for assignments assigned to that student's groups.
- Homework average is total earned marks divided by total possible marks for submitted work.
- Practice count and average are based on submitted StudentPractice records for the student email.
- Assignment topic accuracy uses assignment question conceptTag values.
- Practice topic accuracy uses StudentPractice.topic and total score.
- Weak topics are below 65 percent.
- Strong topics are 65 percent and above.
- Growth timeline combines homework submissions and practice quiz submissions in chronological order.
- Struggle notes include incorrect homework answer AI feedback and low-scoring practice feedback.

Student Reports frontend:
- frontend/src/app/(workspace)/reports/page.tsx
- Fetches summaries with listStudentReports().
- Fetches detail reports with getDetailedStudentReport(email).
- Provides search by student name, email, group, grade, or subject.
- Shows summary metric tiles for registered students, homework average, practice average, and completed grades.
- Shows student scorecards with groups, completion, homework average, and practice count.
- Opens a detailed report modal/panel when a student is clicked.
- The detail overlay is rendered through a document.body portal with z-index 10000 so the top bar cannot overlap it.
- Uses a pure SVG growth line chart with hover tooltips.
- Shows weak and strong topic matrices with progress bars and score colors.
- Shows AI struggle notes.
- Shows tabs for homework logs and practice logs.

Database models
---------------
Assignment:
- Stores generated teacher assessments.
- Includes title, subject, grade, due date, question rows, total marks, optional file/PDF URLs, generation status, job ID, generated result, user ID, and timestamps.
- Generated result contains sections, questions, answer key, school metadata, and total marks.

AssignedAssignment:
- Links an Assignment to a Group.
- Stores due date, hints setting, duration, misconception analysis, hint logs, user ID, and timestamps.

Group:
- Stores teacher class group name, grade, subject, students, rubric, class code, user ID, and timestamps.

StudentCredential:
- Stores registered student name, unique email, group IDs, hashed passcode, and timestamps.

StudentSubmission:
- Stores answers for a student and assigned assignment.
- Includes total score, total marks, start/submission timestamps, auto-submit flag, timer job ID, and per-answer AI feedback.

StudentPractice:
- Stores student-generated AI practice quizzes.
- Includes student email, student name, class code, subject, grade, topic, questions, graded answers, score, total marks, feedback, submittedAt, and timestamps.

LibraryItem:
- Stores library folders and uploaded/exported resources.

Notification:
- Stores teacher activity notifications and read/delete state.

Template:
- Stores reusable assignment/template settings.

User:
- Stores Clerk-linked teacher profile and plan/settings data.

Authentication and permissions
------------------------------
- Teacher workspace routes use Clerk authentication.
- Backend teacher routes generally use requireAuth() and syncUserMiddleware.
- If Clerk environment keys are missing in development, backend injects a mock user ID of user_tester.
- Student-facing routes are separate under /api/student and do not use the teacher auth middleware.
- Report endpoints are teacher-protected and filter all data by teacher-owned group IDs.

API client
----------
frontend/src/lib/api.ts:
- Stores a global Clerk token.
- Waits for Clerk token before teacher API requests.
- Provides teacher API functions for assignments, library, notifications, profile/settings, rubrics, templates, groups, assigned assignments, submissions, and reports.

frontend/src/lib/studentApi.ts:
- Provides student-facing API functions for session, assignments, submissions, uploads, practice generation, practice submission, and practice history.

Background job flow
-------------------
- Teacher creates assignment in frontend.
- Backend stores an assignment with pending/processing status.
- Backend enqueues a BullMQ job.
- Worker parses uploaded source material if present.
- Worker calls Groq with structured prompts.
- Worker updates Assignment result and status.
- WebSocket pushes status updates to the frontend.
- Frontend status page reacts to job progress and routes to result when done.

Design system and UI behavior
-----------------------------
- The teacher workspace uses a glass-style shell with sidebar, top bar, mobile drawer, and bottom mobile nav.
- Framer Motion handles page and modal transitions.
- Tailwind classes define responsive layouts.
- Lucide icons are used for controls and navigation.
- Mobile layout uses a single scrollable main content area inside AppShell.
- The Student Reports detail modal is portaled above the shell chrome to avoid top bar overlap.

Verification notes from Student Reports implementation
------------------------------------------------------
- backend npx tsc --noEmit passed.
- frontend npx tsc --noEmit passed.
- frontend npm run build compiled successfully, then failed during Next page-data collection for an existing workspace route such as /assignments. The new /reports route was emitted in the app manifest. This appears to be a pre-existing Next build/page collection issue rather than a Student Reports type or compilation issue.
- Starting local dev servers from the sandbox was blocked by environment/network permissions: backend could not connect to Redis/MongoDB from the sandbox, and frontend could not bind to port 3000 due EPERM.
